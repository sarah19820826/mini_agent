# 安全审查规则

## 危险函数
- 禁止 `eval()` / `exec()` — 用 `ast.literal_eval()` 替代
- 禁止 `os.system()` — 用 `subprocess.run()` 替代
- `open()` 文件路径需做安全校验

## 凭证管理
- 硬编码密码/密钥 → 用环境变量
- 数据库连接字符串不能写死在代码里
- API Key 不能出现在 git 历史中

## 输入处理
- SQL 查询必须用参数化，禁止字符串拼接
- 用户输入不能直接拼入命令或文件路径
- JSON 解析要处理类型异常

## 日志
- 敏感数据（密码、token）不写日志
- 日志级别正确：error 用于异常，info 用于流程，debug 用于调试
