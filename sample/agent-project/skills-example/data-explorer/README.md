# data-explorer Skill 完整流程

## 目录结构

```
data-explorer/
├── SKILL.md                          ← 操作手册 + MCP工具声明
└── references/
    ├── data-sources.md               ← 数据来源与字段说明
    └── output-format.md              ← 输出格式规范
```

## 完整流程

```
┌─────────────────────────────────────────────────────────┐
│ 第0步：启动 — 扫描 frontmatter                           │
│ 解析 tools: [query_database, query_aggregate, ...]       │
│ → 这9个工具注册到工具列表中                               │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ 第1步：用户输入                                           │
│ "帮我查上个月订单总额"                                     │
│ → LLM 匹配：data-explorer 描述匹配                        │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ 第2步：加载手册 — SKILL.md                                │
│ 手册里写：                                                 │
│   | 工具名 | 用途 | 何时使用 |                             │
│   | query_aggregate | 聚合统计 | 要统计/汇总时 |            │
│                                                              │
│ LLM 判断：用户要统计总额 → 用 query_aggregate               │
│ 不用：query_database, fetch_api, export_csv... (8个)        │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ 第3步：读取资源 — 按需要选择                               │
│ → read_resource("references/data-sources.md")            │
│   了解 orders 表结构：order_id, amount, status            │
│                                                              │
│ 不需要读：output-format.md（用户没要格式化）                 │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ 第4步：执行 — 只调用选中的工具                              │
│ → 调用 query_aggregate 工具                               │
│   查询：SELECT SUM(amount) FROM orders WHERE...          │
│                                                              │
│ 不调用：fetch_api, export_csv, generate_chart (8个)       │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│ 第5步：输出结果                                            │
│ "上个月订单总额为 ¥123,456.78"                             │
└─────────────────────────────────────────────────────────┘
```

## 三种场景对比

同一个 skill，9 个 MCP 工具，不同场景用不同子集：

```
场景A: "查上个月订单总额"
  用 1 个:  query_aggregate
  不用 8 个: 其他全部

场景B: "把JSON文件数据画成图表"
  用 2 个:  parse_json_file → generate_chart
  不用 7 个: 数据库/API相关

场景C: "拉API数据导出CSV"
  用 2 个:  fetch_api → export_csv
  不用 7 个: 数据库/文件相关
```

## 核心设计

**SKILL.md frontmatter 声明所有可用工具**（工具注册层面）
```yaml
tools:
- query_database      ← 注册了
- query_aggregate     ← 注册了
- export_csv          ← 注册了
... 共9个
```

**SKILL.md 正文指导按需使用**（使用层面）
```markdown
| 工具名 | 用途 | 何时使用 |
| query_aggregate | 聚合统计 | 要统计汇总时 |
```

LLM 读了手册后，根据用户需求从 9 个工具中选出最相关的 1-3 个。

## 与 python-review 的对比

| 方面 | python-review | data-explorer |
|---|---|---|
| 按需加载物 | references/ 规则文件 | MCP 工具 |
| 声明方式 | 无 | SKILL.md frontmatter `tools` 字段 |
| 选择逻辑 | 手册表格指导 | 手册表格指导 |
| 共同点 | 都是手册告诉 LLM "什么时候用什么" | 同上 |
